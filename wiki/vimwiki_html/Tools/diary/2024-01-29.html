Magic of parallel programming with CUDA C

In this article, we will discuss how one can utilize CUDA parallel programming in modifying the largest array possible on any computer with NVIDIA GPUs.

We will follow the following steps:

  1. First, find the largest possible array of integers which can be stored in any given computer.
  2. Then we will modify that array using serial programming (C++ standard).
  3. We will then modify that array with parallel programming (CUDA C).

Finding Largest array of integer

We are looking for largest size of array of integers. Any signed/unsigned integer require 4 bytes to be stored. So N such integers will require 4N bytes of memory. To calculate the maximum size of array we need to know the amount of available memory.
Here comes the interesting part, there are two types of memory program can use,

  1. Stack memory
  2. Heap Memory

Stack memory and heap memory are two types of memory used by computer programs for different purposes. They differ in terms of allocation, deallocation, and scope. For more, find Memory allocation

Stack holds limited amount of memory. It varies from computer to computer depending upon OS and compiler from 1MB to 8MB.

Therefore it is better to allocate the memory on run time(dynamic memory allocation), rather than compile time(such memory allocation is automatically handled by stack memory)

We will first make use of stack memory and try to store some array and modify it using some serial program algorithm.
